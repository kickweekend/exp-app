# Exp App — приложение экспертиз (Experteese Lite)

Веб‑приложение на Ruby on Rails 8 с использованием PostgreSQL, демонстрирующее:

- постраничную структуру (контроллеры, вьюхи, маршруты);
- работу с БД Postgres (модели, миграции, связи);
- регистрацию и аутентификацию пользователей;
- сбор и хранение экспертных оценок изображений по темам;
- простое API на Ruby + JavaScript для перелистывания изображений и сохранения оценок;
- локализацию интерфейса на русский и английский языки.

## Технологии

- Ruby 3.x
- Ruby on Rails 8.1
- PostgreSQL
- Минимальный набор гемов: `rails`, `pg`, `puma`, `propshaft`, `bcrypt`, `debug`.

## Подготовка среды

1. Установить PostgreSQL.
2. Создать роль с правами создания БД (аналогично `exp_app_admin`) или использовать роль текущего пользователя.
3. Убедиться, что в `config/database.yml` указаны корректные имя пользователя и пароль (при необходимости раскомментировать `username` / `password`).

## Создание и заполнение базы данных

```bash
bin/rails db:create
bin/rails db:migrate
bin/rails db:seed
```

Скрипт `db/seeds.rb` создаёт:

- пользователя‑демонстратора (`demo@example.com` / пароль `password`);
- 5 тем (город, природа, технологии, еда, животные);
- набор записей изображений, **строго соответствующий файлам в папках тем**.

Структура каталогов с изображениями:

- `public/images/city/*` — городские пейзажи;
- `public/images/nature/*` — природа;
- `public/images/technology/*` — технологии;
- `public/images/food/*` — еда;
- `public/images/animal/*` — животные.

Все файлы (`.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`), лежащие в этих папках,
попадают в базу как отдельные изображения. Количество картинок для оценки
всегда равно текущему числу файлов в соответствующей папке.

Порядок показа изображений определяется **датой создания файла** (если
файловая система не поддерживает birthtime — датой изменения файла).

## Запуск приложения

```bash
bin/rails server
```

Перейдите в браузере:

- `http://localhost:3000/` — корень (по умолчанию локаль `ru`);
- `http://localhost:3000/en` — интерфейс на английском.

## Использование

1. Войти под демо‑пользователем:
   - e‑mail: `demo@example.com`
   - пароль: `password`
   или зарегистрировать нового пользователя на странице регистрации.
2. Выбрать тему из списка.
3. На странице темы:
   - перелистывать изображения кнопками «Предыдущее» / «Следующее»;
   - выставлять оценку (1–5) и комментарий;
   - сохранять результат (через JavaScript‑запрос к API).

### Подготовка собственных изображений

Чтобы избежать геоблокировок и проблем с внешними сервисами, приложение
использует **локальные файлы**:

1. Поместите свои изображения (например, с лицензией CC0 / open‑source) по папкам:
   - `public/images/city`
   - `public/images/nature`
   - `public/images/technology`
   - `public/images/food`
   - `public/images/animal`
2. Имена файлов могут быть любыми; поддерживаются расширения `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`.
3. Выполните `bin/rails db:seed`, чтобы база данных была синхронизирована с содержимым папок.

## Структура функционала

- `app/models` — модели `User`, `Theme`, `Image`, `Evaluation` с валидациями и связями.
- `app/controllers` — контроллеры аутентификации, тем, изображений и оценок, а также API‑контроллеры.
- `app/views` — страницы входа, регистрации, список тем и экран экспертизы изображений.
- `config/locales` — файлы локализации `en.yml` и `ru.yml`.

Код снабжён документирующими комментариями на русском языке в ключевых моделях и контроллерах, чтобы упростить защиту курсового проекта.
